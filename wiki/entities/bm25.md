## Overview

A sparse retrieval algorithm used as a baseline in RLM evaluations, equipping CodeAct agents with a retriever that indexes input context. Represents the class of retrieval-augmented long-context approaches.

## Key Findings

1. For row-wise order-preserving rank and row-wise thresholdable rank are equivalent for binary relevance matrices. (from "On the Theoretical Limitations of Embedding-Based Retrieval")
2. For any fixed embedding dimension d, there exists a binary relevance matrix which cannot be captured via d-dimensional embeddings. (from "On the Theoretical Limitations of Embedding-Based Retrieval")
3. The minimum embedding dimension required to represent a retrieval task is bounded by the sign-rank of the query-relevance matrix. (from "On the Theoretical Limitations of Embedding-Based Retrieval")
4. Token pooling with a pool factor of 3 reduces the total number of stored vectors by 66.7% while retaining 97.8% of the original retrieval performance on average. (from "ColPali: Efficient Document Retrieval with Vision Language Models")
5. Gemini-2.5-Pro (as a reranker/cross-encoder) can solve the LIMIT small task (46 documents, 1000 queries) with 100% accuracy in a single forward pass. (from "On the Theoretical Limitations of Embedding-Based Retrieval")
6. The LIMIT dataset uses a dense qrel matrix with 46 documents and all C(46,2)=1035 top-2 query combinations over a 50k-document corpus. (from "On the Theoretical Limitations of Embedding-Based Retrieval")
7. ColPali architecture extends PaliGemma-3B by adding a projection layer that maps each language model output token embedding to a reduced dimension D=128, enabling ColBERT-style multi-vector representa (from "ColPali: Efficient Document Retrieval with Vision Language Models")
8. ColPali offline document indexing latency is 0.39 seconds per page, compared to 7.22 seconds for a PDF parser pipeline with layout detection, OCR, and captioning. (from "ColPali: Efficient Document Retrieval with Vision Language Models")
9. An RLM initializes a REPL programming environment in which the prompt P is set as the value of a variable, permitting the LLM to write code that peeks into and decomposes P. (from "Recursive Language Models")
10. RLMs treat arbitrarily long user prompts as part of the external environment rather than feeding them directly into the neural network. (from "Recursive Language Models")
11. ColPali achieves an average nDCG@5 of 81.3 on ViDoRe, outperforming the best text-based baseline (Unstructured + Captioning + BGE-M3) which scores 67.0. (from "ColPali: Efficient Document Retrieval with Vision Language Models")
12. State-of-the-art embedding models fail on the LIMIT dataset despite the simple nature of the task, with recall@100 below 20%. (from "On the Theoretical Limitations of Embedding-Based Retrieval")
13. The number of top-k subsets of documents capable of being returned as the result of some query is limited by the dimension of the embedding. (from "On the Theoretical Limitations of Embedding-Based Retrieval")
14. ColPali's training dataset consists of 118,695 query-page pairs: 63% from openly available academic datasets and 37% synthetic, generated from web-crawled PDFs with VLM-generated pseudo-questions. (from "ColPali: Efficient Document Retrieval with Vision Language Models")
15. ColPali's memory footprint is 257.5 KB per page when projecting PaliGemma vectors to dimension 128. (from "ColPali: Efficient Document Retrieval with Vision Language Models")
