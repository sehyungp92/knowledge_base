## Overview

A programming approach where software is built entirely through natural language instructions to an AI, with the programmer ignoring the underlying code entirely. Treats code as free, ephemeral, and disposable.

## Key Findings

1. Agent.ai has 1.3 million users, with 3,000 people having built agents and approximately 1,000 agents published on the platform. (from "The Agent Network — Dharmesh Shah, Agent.ai + CTO of HubSpot")
2. The author coined the term 'vibe coding' in a tweet and was surprised by how widely it spread. (from "2025 LLM Year in Review")
3. Current AI agents lack continual learning: they cannot persistently retain new information told to them. (from "Andrej Karpathy — “We’re summoning ghosts, not building animals”")
4. Software 3.0 is a new programming paradigm where LLMs are programmed via prompts written in natural language (English). (from "Andrej Karpathy: Software Is Changing (Again)")
5. Every agent built on Agent.ai automatically gets a REST API and automatically appears in the MCP server, enabling invocation in any form the developer chooses. (from "The Agent Network — Dharmesh Shah, Agent.ai + CTO of HubSpot")
6. Software 1.0 is traditional code written by humans for computers; Software 2.0 is neural network weights produced by optimizing over datasets rather than being written directly. (from "Andrej Karpathy: Software Is Changing (Again)")
7. Current AI agents are insufficiently multimodal and lack robust computer use capabilities. (from "Andrej Karpathy — “We’re summoning ghosts, not building animals”")
8. Reinforcement Learning from Verifiable Rewards (RLVR) emerged as the de facto new major stage in LLM training in 2025, added on top of the existing pretraining, SFT, and RLHF pipeline. (from "2025 LLM Year in Review")
9. LLM judges used as reward functions in reinforcement learning are vulnerable to adversarial examples that cause nonsensical outputs to receive maximum reward. (from "Andrej Karpathy — “We’re summoning ghosts, not building animals”")
10. RLVR introduced test-time compute scaling as a new knob for capability, allowing models to generate longer reasoning traces and increase 'thinking time'. (from "2025 LLM Year in Review")
11. Paid provides billing, invoicing, monetization, margin management, collections, revenue recognition, and vendor management infrastructure for AI agent companies. (from "Pricing in the AI Era: From Inputs to Outcomes, with Paid CEO Manny Medina")
12. "Work as a Service" is a business model where software performs the work (replacing the human doing it), while "Results as a Service" charges for outcomes rather than the work performed. (from "The Agent Network — Dharmesh Shah, Agent.ai + CTO of HubSpot")
13. Cursor revealed a distinct new layer of LLM applications that bundle and orchestrate LLM calls for specific verticals, doing context engineering, multi-call DAG orchestration, application-specific GUI (from "2025 LLM Year in Review")
14. Process-based supervision faces a credit assignment problem: it is unclear how to automatically assign partial credit to intermediate steps in a solution. (from "Andrej Karpathy — “We’re summoning ghosts, not building animals”")
15. LLMs are trained through imitation of human-generated internet data rather than evolutionary processes, making them qualitatively different from biological intelligences. (from "Andrej Karpathy — “We’re summoning ghosts, not building animals”")
