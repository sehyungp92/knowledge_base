### Motivation & Prior Limitations
- RL-trained LLM agents (via RLHF or RLVR) are fundamentally bottlenecked by their dependence on large, high-quality, human-curated datasets, which is time-consuming, costly, and tethers AI capability to the ceiling of human knowledge and annotation speed.
  - This scalability bottleneck is a hard limit: the system cannot discover reasoning strategies beyond what human annotators can label or verify.
- Existing self-play and self-evolution frameworks, while promising as data-free alternatives, suffer from two compounding failures: their generated curricula rarely surpass the model's current complexity (because they are bounded by its inherent knowledge), and they operate only in single-round interactions.
  - Methods like Absolute Zero and R-Zero demonstrate this ceiling — tasks generated by the model cannot exceed the model's own capability horizon, leading to curriculum stagnation and learning saturation.
  - Single-turn interaction formats fail to capture the dynamic, context-dependent problem-solving that tool use and multi-step reasoning require, leaving these skills unlearnable within existing frameworks.

---

### Proposed Approach
- Agent0 is a fully autonomous co-evolutionary framework that bootstraps two functionally distinct agents — a Curriculum Agent (πθ) and an Executor Agent (πϕ) — both initialized from the same base LLM, with no external data or human annotations at any stage.
  - Unlike prior self-play methods that use a single model in a single role, Agent0 separates task generation and task solving into two continuously co-evolving agents that mutually pressure each other, creating a symbiotic competitive dynamic analogous to expert iteration but without a ground-truth oracle.
  - The Curriculum Agent is trained via GRPO to maximise a composite reward RC that incentivises generating tasks at the Executor's capability frontier: an Uncertainty Reward (Runc, maximised when Executor self-consistency ˆp ≈ 0.5, indicating neither trivially easy nor impossible), a Tool Use Reward (Rtool, proportional to the number of code interpreter invocations by the Executor, capped at C=4 to prevent reward hacking), and a Repetition Penalty (Rrep, using BLEU-based cluster sizes to enforce curriculum diversity within each training batch).
- The Executor Agent is trained on a filtered dataset D(t) containing only tasks with ˆp ∈ [0.3, 0.8] (controlled by threshold δ=0.25), using majority-vote pseudo-labels as supervision in the absence of ground-truth answers, via a novel algorithm called Ambiguity-Dynamic Policy Optimization (ADPO).
  - ADPO introduces two modifications to standard GRPO: ambiguity-aware advantage scaling, which down-weights gradient updates from high-ambiguity (low-ˆp) samples where pseudo-labels are least reliable, and ambiguity-modulated trust regions, which dynamically relax the upper PPO clipping bound ϵhigh(x) for ambiguous samples to permit larger gradient steps that can surface low-probability but correct reasoning paths — addressing the observed empirical failure where standard clipping disproportionately suppresses unlikely tokens.
- A sandboxed code interpreter is integrated into the Executor's multi-turn rollout, enabling an interleaved cycle of text reasoning, Python code emission, execution feedback, and iterative self-correction — and this tool integration drives the core virtuous cycle: a stronger Executor compels the tool-equipped Curriculum Agent to generate harder, more tool-reliant tasks.

### Results & Capabilities
- Agent0 applied to Qwen3-8B-Base achieves a 58.2 average score on mathematical reasoning benchmarks, outperforming R-Zero by 6.4%, Absolute Zero by 10.6%, and Socratic-Zero (which uses external OpenAI APIs) by 3.7% — all without any human-annotated data.
  - On individual benchmarks, Agent0 reaches 62.4 on AMC, 61.3 on Minerva, 82.4 on MATH, 94.5 on GSM8K, 54.0 on Olympiad-Bench, 24.8 on AIME24, and 28.0 on AIME25.
- Capabilities generalise substantially beyond mathematics: on Qwen3-8B-Base, Agent0 achieves 42.1 overall average on general-domain reasoning (SuperGPQA, MMLU-Pro, BBEH), outperforming all compared data-free methods including Socratic-Zero (39.2) and Absolute Zero (39.9).
  - This represents an 18% improvement in mathematical reasoning and 24% improvement in general reasoning over the base model, demonstrating that tool-augmented multi-step reasoning transfers to domains not seen during training.
- Ablation studies confirm that every component is load-bearing: removing Curriculum Agent training drops average performance by 9.3%, removing the Tool Use Reward drops it by 7.2%, removing the Repetition Penalty degrades it by 7.6%, replacing ADPO with standard GRPO costs 1.9%, and removing multi-turn reasoning costs 2.7%.
- Quantitative analysis of curriculum evolution confirms the theoretical mechanism: fixing the Executor at Iteration 1 and evaluating it on tasks generated by later curriculum agents shows pass rates declining from 64.0% (Iter 1 tasks) to 58.5% (Iter 2 tasks) to 51.0% (Iter 3 tasks), while average tool calls per task increase from 1.65 to 2.10 to 2.60 — directly validating the self-reinforcing complexity spiral.
- Qualitative case analysis shows the Curriculum Agent progresses from basic geometry problems in Iteration 1 to complex constraint satisfaction and combinatorial tasks by Iteration 3, while the Executor demonstrates reliable hybrid reasoning — combining natural language pattern identification with Python code verification within a single trajectory.

## Key Claims

1. Agent0 improves Qwen3-8B-Base by 18% on mathematical reasoning benchmarks without any external data
2. Agent0 improves Qwen3-8B-Base by 24% on general reasoning benchmarks without any external data
3. Existing self-evolution frameworks are capped by the model's inherent knowledge, causing generated tasks to rarely surpass the model's current complexity
4. Existing self-play frameworks typically operate only in single-round interactions, failing to capture dynamic, context-dependent real-world problems
5. Agent0 initializes both the curriculum agent and executor agent from the same base LLM, then co-evolves them through symbiotic competition
6. The curriculum agent's uncertainty reward is maximized when the executor's self-consistency is 0.5, penalizing tasks that are either too easy or too hard
7. The tool use reward is capped at a maximum number of tool calls (C=4) to prevent rewarding excessive or spurious tool use
8. A repetition penalty using BLEU-based clustering is applied to encourage diversity within training batches
9. Executor agent training filters tasks to those with self-consistency scores within δ=0.25 of 0.5, retaining only tasks at the capability frontier
10. Majority voting on k=10 executor responses is used to derive pseudo-labels for reward computation without any ground-truth labels

## Capabilities

- Dual-agent co-evolutionary framework (Agent0) can train high-performing LLM agents from scratch without any human-curated data — a curriculum agent generates progressively harder tasks while an executor agent learns to solve them, driven entirely by RL rewards based on executor uncertainty and tool-
- Tool-reward incentivisation in curriculum RL training drives autonomous escalation of task complexity — explicitly rewarding task generation that elicits tool use causes the curriculum agent to progressively increase average tool calls per task (1.65 → 2.10 → 2.60 across iterations) without any huma
- Ambiguity-Dynamic Policy Optimization (ADPO) improves RL training on pseudo-labeled data by dynamically scaling advantage weights and trust region bounds based on majority-vote self-consistency — downweighting unreliable high-ambiguity samples and relaxing upper clipping bounds for them to allow exp
- Multi-turn tool-integrated rollout during RL training enables agents to iteratively interleave natural language reasoning with code execution and error correction within a single trajectory, producing hybrid reasoning chains that mimic self-correction
- Mathematical reasoning skills acquired via zero-data self-evolution transfer to general-domain reasoning — Agent0 trained only on self-generated math tasks achieves state-of-the-art results on SuperGPQA, MMLU-Pro, and BBEH without any general-domain training

## Limitations

- Zero-data self-evolving frameworks without external verifiable tools inevitably hit a capability ceiling — the curriculum and executor are both bounded by the model's inherent knowledge, causing generated tasks to rarely exceed current model complexity and producing curriculum stagnation or mode col
- Agent0's self-evolution is structurally dependent on having a reliable sandboxed external tool (code interpreter) — the virtuous cycle only exists because the tool provides objective, execution-grounded feedback; the approach cannot extend to domains lacking verifiable external tools
- Pseudo-label noise from majority voting corrupts training signal for high-ambiguity tasks — when self-consistency ˆp(x) is low, the majority answer is prone to errors and standard RL training risks reinforcing incorrect reasoning
- Standard GRPO's static upper clipping bound disproportionately suppresses low-probability token exploration — empirically, the upper clipping bound is predominantly triggered by tokens with low probabilities, preventing emergence of novel reasoning paths
- Zero RL (RL without supervised fine-tuning grounding) in multi-turn scenarios is notoriously unstable — the field has identified this as a persistent challenge requiring dedicated mitigation techniques
- Agent0 is only evaluated on closed-form reasoning benchmarks (math and general QA) — there is no evaluation on real-world agentic tasks, computer use, long-horizon planning, or environments without ground-truth answer verification
- Only 3 co-evolutionary iterations are evaluated — long-term training stability, potential reward hacking, and whether improvement continues past iteration 3 or plateaus are entirely unknown
- Reward hacking risk in tool-use incentivisation requires an explicit cap (C=4) on rewarded tool calls — without this, the curriculum agent would be incentivised to generate spurious, tool-heavy tasks rather than genuinely complex ones
- The paper is self-described as 'Preliminary work' — results have not been validated at scale, across diverse domains, or in production-grade settings
- RLHF and RLVR methods remain heavily dependent on massive high-quality human-curated datasets — this is a structural bottleneck that Agent0 addresses only for verifiable reasoning domains, leaving the broader agent training pipeline still data-constrained

## Bottlenecks

- Self-evolving LLM training frameworks are capability-bounded by the base model's inherent knowledge — without external grounding, the curriculum and executor share the same knowledge space, causing generated tasks to stagnate at the model's current complexity frontier and blocking further self-impro
- Human-curated training data creates a severe scalability bottleneck for RL-trained agents — annotation is time-consuming, labor-intensive, and costly, and fundamentally limits AI capability growth to the rate of human annotation and the boundary of existing human knowledge
- Existing self-play frameworks operate in single-round interactions only, preventing generation of context-dependent, multi-turn curricula that reflect real-world problem complexity — agents cannot learn skills requiring dynamic context management or sequential tool use
