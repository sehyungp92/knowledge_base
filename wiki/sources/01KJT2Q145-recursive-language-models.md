2025-12-31 · paper · Alex L. Zhang, Tim Kraska, Omar Khattab

### Motivation & Prior Limitations
- Frontier LLMs suffer from hard context window ceilings and context rot — a phenomenon where quality degrades steeply as prompts grow longer, even within the nominal window limit.
  - GPT-5's 272K token context window is insufficient for long-horizon tasks requiring tens or hundreds of millions of tokens, and its performance on OOLONG (linear-complexity) degrades faster than on NIAH (constant-complexity), confirming that effective context length is task-dependent.
- Context compaction (iterative summarization) is the dominant general-purpose workaround, but it is lossy by design and fails on tasks requiring dense access throughout the prompt because it presumes early details can be safely discarded.
  - Summary agents achieve 70.5% on BrowseComp-Plus but collapse to near-zero on OOLONG-Pairs (F1 ≈ 0.1), where quadratic pairwise aggregation cannot be approximated by summarization.
- Prior coding agents and retrieval agents treat external data as an environment but still load user prompts directly into the LLM context window, so they inherit the base model's window limit and cannot process inputs beyond it.
- Prior self-delegation approaches (sub-agent architectures) are bottlenecked by output length limits because sub-calls are verbalized autoregressively rather than generated programmatically inside loops.

---

### Proposed Approach
- Recursive Language Models (RLMs) treat the user prompt as an external environment variable rather than a direct neural input, enabling a base LLM to programmatically examine, decompose, and recursively call itself over arbitrary slices of the prompt via a persistent Python REPL.
  - The prompt P is stored as a variable in a Read-Eval-Print Loop (REPL) environment; the LLM receives only constant-size metadata (length, short prefix, access instructions) in its actual context window, never the full prompt.
  - The LLM writes executable code that can slice, filter, and transform P using regex or chunking, launch recursive sub-RLM calls inside arbitrarily large loops, and accumulate results into REPL variables — enabling Ω(|P|) or Ω(|P|²) semantic work without window overflow.
- Three design choices distinguish RLMs from superficially similar scaffolds: (1) the prompt lives in the environment, not the context window; (2) outputs are built into REPL variables rather than generated autoregressively to a Finish action; and (3) sub-calls are invoked programmatically inside code loops rather than verbalized one at a time.
- For post-training, RLM-Qwen3-8B was fine-tuned on only 1,000 filtered RLM trajectories generated by Qwen3-Coder-480B-A35B on LongBenchPro tasks, exploiting the insight that effective sub-call behavior generalizes from short-horizon reasoning to long-horizon recursive decomposition.

---

### Results & Capabilities
- RLMs substantially outperform base frontier models and all task-agnostic scaffolds across four diverse long-context benchmarks, often by double-digit margins, while maintaining comparable or lower median API cost.
  - On BrowseComp-Plus (1K), RLM(GPT-5) achieves 91.3% versus 70.5% for the best baseline (summary agent) and 0% for the base GPT-5 (context overflow); average RLM cost is $0.99 versus a linearized baseline cost of $1.50–$2.75 for direct ingestion.
  - On OOLONG-Pairs (quadratic-complexity), RLM(GPT-5) reaches F1 58.0% while GPT-5 base scores 0.1% and all other baselines score below 30%, demonstrating an emergent capability absent in the base model even within its context window.
- RLM performance degrades significantly more slowly than base model performance as input length increases, with the gap widening for higher-complexity tasks; RLMs consistently outperform GPT-5 on all three scaling benchmarks beyond context length 2¹⁴.
- The REPL environment alone (without recursive sub-calls) is sufficient to handle inputs beyond the context window, but recursive sub-calling provides an additional 10–59% improvement on information-dense tasks where semantic transformation of every input line is required.
- A small fine-tuned model, RLM-Qwen3-8B, outperforms vanilla Qwen3-8B by 28.3% on average across all four evaluation tasks and approaches GPT-5 quality on three of them, demonstrating rapid transferable gains from minimal domain-specific training data.
- RLMs are model-agnostic but exhibit model-dependent behavior: RLM(GPT-5) is conservative with sub-calls and nearly solves all BrowseComp-Plus tasks (91.3%), while RLM(Qwen3-Coder) launches sub-calls per line and achieves only 44.7%, despite an explicit system-prompt warning against overuse.

### Implications
- RLMs reframe long-context scaling as an inference-time compute problem rather than an architecture or training problem, suggesting that the effective context horizon of any LLM can be extended by orders of magnitude without retraining, purely through scaffold design.
- The paradigm establishes symbolic recursion — programmatic self-invocation inside a code environment — as a qualitatively new capability tier for LLMs, bridging the gap between neural attention (bounded) and symbolic computation (unbounded), with implications for long-horizon agentic tasks like deep research, codebase reasoning, and multi-document aggregation.
- The rapid generalization of RLM fine-tuning (28.3% improvement from 1,000 cross-domain samples) suggests that recursive reasoning may constitute a learnable, transferable skill analogous to chain-of-thought reasoning, opening a potential new axis of scale beyond parameter count and context length.
- The explicit demonstration that task complexity (constant vs. linear vs. quadratic processing complexity) governs effective context length has implications for benchmark design: single-point context-length evaluations like RULER/NIAH underestimate real-world degradation for aggregation-hea

## Key Claims

1. Recursive Language Models can successfully process inputs up to two orders of magnitude beyond model context windows.
2. Frontier reasoning models exhibit context rot, where quality degrades steeply as prompts get longer.
3. Context compaction (summarization) is rarely expressive enough for tasks that require dense access throughout the prompt.
4. RLMs treat arbitrarily long user prompts as part of the external environment rather than feeding them directly into the neural network.
5. An RLM initializes a REPL programming environment in which the prompt P is set as the value of a variable, permitting the LLM to write code that peeks into and decomposes P.
6. Prior self-delegation approaches are handicapped by the underlying LLM's limited output lengths because they verbalize sub-calls autoregressively rather than producing them programmatically.
7. RLM-Qwen3-8B, post-trained with only 1,000 samples, outperforms the base Qwen3-8B model by 28.3% on average across four long-context tasks.
8. The effective context window of an LLM cannot be understood independently of the specific task; more complex problems exhibit degradation at shorter context lengths.
9. Frontier models can reliably solve needle-in-a-haystack tasks at 1M+ tokens but struggle at far shorter lengths on OOLONG, where the answer depends on nearly every line.

## Capabilities

- Recursive Language Models (RLMs) enable LLMs to process inputs up to 10M+ tokens — two orders of magnitude beyond their native context windows — by treating the prompt as an external environment variable that the model interacts with symbolically via a REPL, without ever loading the full prompt into
- RLMs outperform context compaction, BM25 retrieval agents, and code execution agents on all tested long-context benchmarks by up to 2× in accuracy while maintaining comparable or lower median API cost — without task-specific engineering.
- Frontier LLMs can be prompted — without any fine-tuning — to behave as RLMs: decomposing arbitrarily long inputs via symbolic recursion, filtering context with code, and launching programmatic sub-calls over slices of the input.
- An 8B model (RLM-Qwen3-8B) fine-tuned on just 1,000 RLM trajectories from an unrelated domain improves performance by a median of 28.3% over the base Qwen3-8B RLM baseline and approaches vanilla GPT-5 quality on three long-context tasks — demonstrating extreme sample efficiency for recursive capabil
- RLMs can generate effectively unbounded output token sequences by accumulating sub-LM call results in REPL variables and stitching them programmatically, bypassing the base LM's output length limit.

## Limitations

- RLM performance is slightly worse than the base LM on short inputs — the overhead of REPL initialization and symbolic interaction imposes a crossover penalty, meaning RLMs should only be invoked above a threshold input length.
- Current RLM implementations use synchronous, blocking LM calls — all sub-calls run sequentially, not in parallel — creating significant inference latency proportional to recursion depth and making real-time or latency-sensitive deployment impractical.
- RLMs exhibit high cost variance — median costs are comparable to or cheaper than base model calls, but tail runs can be significantly more expensive due to unpredictable trajectory lengths, creating budget uncertainty for production deployment.
- RLM decomposition strategies are primitive — observed RLMs only use uniform chunking or keyword-based regex filtering, not principled hierarchical or semantic partitioning, which likely limits performance on complex information-dense tasks.
- RLM effectiveness is highly dependent on base model quality — GPT-5 RLM achieves 91.3% on BrowseComp-Plus while Qwen3-Coder RLM achieves only 44.7%, and Qwen3-8B RLM achieves only 2.0%, making the paradigm a capability amplifier rather than a capability equalizer.
- Current evaluation is restricted to four synthetic long-context benchmarks with controlled corpora; evaluation on naturalistic, open-ended long-context tasks (e.g., real research, long-horizon agent workflows) is entirely absent.
- Recursion depth in current RLMs is limited to one level (sub-calls are plain LM calls, not themselves RLMs) — deeper hierarchical recursion that could handle even larger inputs or more complex nested decompositions has not been explored.
- Frontier LLMs fail catastrophically on quadratic-complexity long-context tasks — GPT-5 achieves <0.1% F1 on OOLONG-Pairs with its native context window, and all non-RLM baselines including summary agents also score near zero — indicating a fundamental processing limit for information-dense pairwise 
- No sandboxed REPL execution is used in current RLM implementations — code generated by the LM runs in an unsecured Python environment, raising significant security and reliability concerns for any deployment beyond research settings.
- Context compaction/summarization — the dominant deployed approach to long-context management — is fundamentally inadequate for tasks requiring dense access to all parts of a long prompt, not merely an engineering limitation but an inherent expressiveness gap.
- RLM sub-call behavior is model-dependent and unpredictable — Qwen3-Coder launches one sub-call per line (extremely granular) while GPT-5 is conservative, requiring model-specific prompt tuning to control sub-call frequency and cost.

## Bottlenecks

- Synchronous, sequential recursive LM call execution creates a hard latency ceiling for RLM deployment — each level of recursion multiplies wall-clock time, making deep or wide decompositions impractical for latency-sensitive applications.
- No principled method for adaptive context decomposition in RLMs — models default to uniform chunking or keyword heuristics rather than semantically-informed partitioning, capping the quality ceiling for information-dense tasks.
- Training data and curricula for natively recursive LMs do not exist at scale — the demonstrated approach uses 1,000 samples from unrelated domains, suggesting the full training potential of RLMs is largely unexplored and blocked by data scarcity.
