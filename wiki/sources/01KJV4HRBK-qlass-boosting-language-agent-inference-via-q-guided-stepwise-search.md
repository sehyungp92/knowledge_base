### Motivation & Prior Limitations
- Most language agent systems rely on outcome reward models (ORMs) that assign a single scalar reward to an entire trajectory, which is insufficient for guiding multi-step reasoning in complex interactive environments.
  - A trajectory that achieves a high final reward does not guarantee that every intermediate action was optimal — agents may complete tasks through inefficient or cyclic paths that ORMs cannot diagnose.
  - Back-and-forth agent behaviors (e.g., repetitive environment queries, cyclic reasoning loops) create stepwise inefficiencies that sparse outcome rewards cannot detect or penalize.
- Existing process reward models (PRMs) that do provide step-level feedback require either costly human annotation (Lightman et al., 2023; Uesato et al., 2022) or computationally heavy Monte Carlo rollouts (Math-Shepherd), making them impractical for self-improvement of open language agents.
- Collecting expert trajectory data for agent tasks is a critical bottleneck due to the inherent complexity and diversity of real-world interactions, limiting both performance and scalability of supervised fine-tuning approaches.
- Recent large-scale systems including KIMI-k1.5 and DeepSeek-R1 have reported failures with process reward modeling, suggesting the field has not yet resolved how to make PRMs reliable for agent tasks — QLASS directly contests this pessimism for the agentic setting.

---

### Proposed Approach
- QLASS (Q-guided Language Agent Stepwise Search) adapts Q-learning from classical RL to open language agents by using an exploration tree structure to extract Q-values offline, bypassing the instability and prohibitive sample costs of direct online RL in high-dimensional language action spaces.
  - Rather than learning Q-values via online interaction, QLASS builds an exploration tree per task from trajectories generated by an SFT-initialized policy, then propagates outcome rewards backward through the tree using the Bellman equation: Q(s_t, a_t) = r_t + γ · max_{a_{t+1} ∈ C_t} Q(s_{t+1}, a_{t+1}).
  - This produces a supervised dataset of (state, action, Q-value) triples without requiring any human annotation of intermediate steps — the only ground-truth signal needed is the final environment reward.
- A Q-network (QNet) is then trained from an SFT-initialized LLM backbone using MSE regression on these estimated Q-values, producing a function approximator that scores the long-term utility of any partial solution at inference time.
- At inference, Q-guided generation samples multiple candidate actions at each step and executes the one with the highest QNet score, steering agent behavior toward higher long-term value without requiring beam search or MCTS over the full action space.
- Tree pruning strategies reduce construction cost: expansion is limited to early trajectory steps (first 3–5 steps depending on environment complexity), and branches terminating with zero outcome reward propagate a Stop expansion signal back to the earliest unexpanded node, prioritizing generation budget toward promising trajectories.
- Unlike Putta et al. (2024), QLASS does not rely on a proprietary LLM for reward scoring; unlike Wang et al. (2024) and Zhai et al. (2024), it targets environments with larger search spaces and deeper search depths; and unlike ReST-MCTS* (Zhang et al., 2024), it uses a simpler single-stage training pipeline.

### Results & Capabilities
- QLASS achieves significant performance improvement over SFT baselines and outcome-reward-guided methods on three diverse interactive agent benchmarks: WebShop (web-based product search, ~5 turns), ALFWorld (embodied household tasks, ~10 turns), and SciWorld (scientific reasoning, ~14 turns).
  - These environments all provide only a single sparse outcome reward, making them a direct test of whether stepwise Q-value guidance adds value beyond outcome-level signals.
- QLASS retains strong performance even when annotated training data is reduced by nearly half, demonstrating that Q-value-based process supervision is more data-efficient than approaches dependent on dense expert annotation.
  - This data efficiency is attributed to the Bellman propagation mechanism, which generates rich intermediate supervision from limited endpoint signals rather than requiring labeled intermediate steps.
- Qualitative analysis confirms that Q-guided generation leads to more targeted decision-making, with agents visibly avoiding repetitive or cyclic actions that SFT and ORM-guided baselines fall into.

---

### Implications
- Adapting Q-learning to language agents via offline tree-based Q-value extraction offers a practical middle path between purely supervised imitation learning and full online RL, enabling process reward modeling without human annotation or expensive rollout budgets — a potentially important scaling direction for open-source agent development.
- The result directly challenges the emerging consensus (from KIMI-k1.5 and DeepSeek-R1) that process reward modeling is unreliable: QLASS argues PRMs are not merely unreliable but have been poorly adapted for the agentic setting, where stepwise inefficiencies are structurally different from those in math reasoning chains.
- If the approach generalizes, it suggests that inference-time compute scaling for agents should be guided by Q-value estimates rather than outcome-only verifiers, which has implications for how test-time search budgets are allocated in long-horizon autonomous agent pipelines.
- The tree-based Q-value extraction paradigm connects classical RL value function theory with LLM self-improvement, potentially opening a bridge between MCTS-based planning research and LLM agent training without requiring MCTS at inference time.

## Key Claims

1. Most existing works use an outcome reward model to optimize policies across entire trajectories due to lack of annotations of intermediate interactions, which may lead to sub-optimal policies.
2. QLASS retains strong performance even when nearly half of the annotated training data is removed.
3. A trajectory achieving a high final outcome reward does not necessarily indicate that every action taken was optimal; some actions could have been inefficient or suboptimal.
4. An outcome-reward model cannot accurately score each step within a long trajectory in intricate scenarios.
5. Existing process reward modeling methods rely on either costly human annotation or computationally heavy random rollouts, rendering them inefficient for self-improvement of language model agents.
6. KIMI-k1.5 and Deepseek-R1 report failures in process reward modeling.
7. Process reward modeling is indispensable for agent tasks because back-and-forth agent behaviors inherently create stepwise inefficiencies that sparse outcome rewards cannot diagnose.
8. QLASS employs the Bellman equation to learn a Q-based process reward that captures how immediate decisions contribute to longer-term payoffs, enabling finer-grained control over reasoning trajectories
9. Scaling up test-time computation is a promising approach for LLMs to improve their outputs more intelligently.
10. Reward models are an essential component of inference-time scaling methods as they evaluate the quality of self-explored data.

## Capabilities

- Automatic Q-value estimation for language agent process reward modeling using self-generated exploration trees and Bellman equation — eliminates need for human-annotated step-level labels
- Q-guided stepwise inference for language agents: a trained QNet scores multiple sampled actions at each step, enabling the agent to select the highest long-term value action during test-time
- Language agent self-improvement with significantly reduced annotation requirements — process reward model training retains strong performance with roughly 50% less labeled data

## Limitations

- Outcome-reward models cannot accurately score individual steps within long agent trajectories — a high final reward does not imply intermediate actions were optimal
- Direct Q-learning adaptation to language agents is sample-inefficient due to vocabulary-scale action spaces that cause combinatorial explosion of exploration paths
- Online Q-learning in language settings is impractical — unbounded vocabulary and sparse rewards make real-time value updates unstable and prohibitively expensive
- Human annotation of intermediate agent interaction steps is prohibitively expensive — creates a fundamental data bottleneck for scaling process reward models to complex multi-turn agent tasks
- Exploration tree construction for deep agent trajectories is computationally expensive — expanding all potential tree nodes without pruning would be prohibitive for real-world tasks
- Most language agent environments provide only sparse terminal rewards with zero intermediate signal — making per-step credit assignment structurally impossible without learned intermediaries
- Process reward modeling has been reported to fail by KIMI-k1.5 and DeepSeek-R1 for general LLM tasks — indicating the technique has a domain-specific applicability boundary
- Back-and-forth agent interaction patterns create stepwise inefficiencies — repetitive environment queries and cyclic reasoning loops that sparse outcome rewards cannot diagnose or penalize during training
- QLASS is validated only on simulated benchmark environments with clean terminal signals (WebShop, ALFWorld, SciWorld) — no evidence of generalization to open-domain or real-world deployment contexts
- Existing process reward model construction methods require either costly human step-level annotation or computationally heavy random rollouts — neither approach scales to the diversity of real-world agent tasks

## Bottlenecks

- Language agent environments provide only terminal outcome rewards with zero intermediate signal — per-step credit assignment requires either expensive human annotation or learned value approximation, blocking scalable agent RL
- Vocabulary-scale action spaces make direct RL exploration intractable for language agents — the combinatorial explosion of possible action sequences requires indirect approximations rather than standard online RL
- Human annotation cost for intermediate agent interaction steps creates a data bottleneck — complex multi-turn agent tasks require labeling diverse real-world interaction sequences, limiting dataset scale and task coverage

## Breakthroughs

- QLASS: automated Q-value estimation from self-generated exploration trees enables annotation-free process reward modeling for language agents — Bellman backup propagates terminal environment rewards to all intermediate steps
